# Contributing to REPLaiTE

Thank you for considering contributing to REPLaiTE! We aim to create a friendly and welcoming community. To help you get started, here are some guidelines for contributing to our iOS Swift project.

## How to Contribute

1. **Fork the Repository**: Start by forking our repository and cloning it to your local machine.
2. **Create a New Branch**: Create a new branch for your feature or bugfix. For example:
   ```bash
   git checkout -b feature/YourFeatureName
   ```
3. **Make Your Changes**: Implement your feature or fix the bug. Be sure to write clear and maintainable code.
4. **Test Your Changes**: Please ensure that your changes are well-tested. We use XCTest framework for testing in Swift.
5. **Commit Your Changes**: Commit your changes with a clear message explaining your reasoning. For example:
   ```bash
   git commit -m 'Add new feature'
   ```
6. **Push to Your Fork**: Push your changes to your forked repository:
   ```bash
   git push origin feature/YourFeatureName
   ```
7. **Create a Pull Request**: Navigate to the original repository, and create a pull request (PR) from your fork's branch. Include a description of your changes and any relevant issues it addresses.
